# Redrob AI Hackathon — Track 1: Intelligent Candidate Ranking

## Overview

This solution ranks 100,000 synthetic candidate profiles for a **Senior AI Engineer (Founding Team)** role at Redrob AI. Instead of keyword matching, it uses a multi-signal semantic scoring pipeline that evaluates **career trajectory, behavioral engagement, and honeypot detection**.

## Architecture

```
candidates.jsonl (100K)
       ↓
  precompute.py  [Phase 1 — no time limit]
       ↓
  features.parquet  ← saved to disk
       ↓
  rank.py  [Phase 2 — <5 min CPU]
       ↓
  submission.csv  (top 100 with scores & reasoning)
```

## Key Design Decisions

### Why NOT keyword matching
The JD explicitly warns against it. Keyword stuffers (e.g. a Marketing Manager with "RAG, Pinecone, LLM" in skills) would rank #1 with naive TF-IDF-only scoring. Our system detects and penalizes these patterns.

### Scoring Formula
```
final_score = (skill_match    × 0.40)
            + (career_quality × 0.25)
            + (behavioral     × 0.20)
            + (availability   × 0.10)
            + (location_fit   × 0.05)
            - (disqualifier_penalty × 0.30)
            + (education_bonus × 0.05)
```

**Skill Match** (TF-IDF cosine + hard skill count + assessment + GitHub):  
- Blends semantic similarity with explicit JD keyword coverage
- Proficiency-weighted: expert skills count 3×, advanced 2×

**Career Quality**:  
- % of career at product companies vs. consulting farms
- Years spent in applied ML/AI roles
- Job-hopping penalty (avg tenure < 18 months)

**Behavioral**:  
- Recruiter response rate (35%), recency (30%), open-to-work (20%), interview completion (15%)

**Honeypot Detection** (caps score at 0.05):  
1. Career duration impossibility (total months > YoE×12 + 18)  
2. Expert skill with zero duration months  
3. Non-technical title + >5 AI skills  
4. All skills listed as "expert"
5. Too-many-skills for very low YoE

## Reproduce (single command)

To run end-to-end (Phase 1 + Phase 2):
```bash
python rank.py --candidates "data/[PUB] India_runs_data_and_ai_challenge/India_runs_data_and_ai_challenge/candidates.jsonl" --out ./submission.csv
```

Or separately:
```bash
# Phase 1: Precompute features (~4 minutes on CPU)
python precompute.py --candidates "data/[PUB] India_runs_data_and_ai_challenge/India_runs_data_and_ai_challenge/candidates.jsonl" --out ./features.parquet

# Phase 2: Rank candidates (<5 seconds on CPU)
python src/ranker.py --features ./features.parquet --out ./submission.csv

# Validate final submission format and rules
python "data/[PUB] India_runs_data_and_ai_challenge/India_runs_data_and_ai_challenge/validate_submission.py" submission.csv
```
*Note: Always use `-X utf8` flag on Windows (`python -X utf8 ...`) to prevent console encoding crashes.*

## Evaluation Alignment

The judges score with:  
`Final = 0.50×NDCG@10 + 0.30×NDCG@50 + 0.15×MAP + 0.05×P@10`

**NDCG@10 is 50% of score** — so we prioritize getting the top 10 exactly right over optimizing the tail.

## AI Tools Declaration

- Feature extraction: `scikit-learn TfidfVectorizer` (CPU, no API calls)
- Scoring: `numpy` / `pandas` (pure math, microseconds per candidate)
- Reasoning strings: Generated by rule-based template from candidate features (no LLM at inference time, ensuring < 5-min reproducibility)
